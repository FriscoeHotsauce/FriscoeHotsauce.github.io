Work Experience

My Teams:

  • NIADS group:
    One of Iowa State's network administration groups. This is where I got started at Iowa State.
  • ITS Java Dev:
    I transitioned to this team after some restructuring at ISU, and ended up working on multiple projects within ISU.

Nature of the work:

  • NIADS:
    We worked on legacy systems, primarily maintaining "Account Services on the Web" or ASW. The service registers students with a netid at iowa state, manages Active Directory groups for everyone faculty and staff, and also handles a miscellaneous assortment of other tasks the university requires.
  • ITS Java Dev:
    I developed full stack features for different web applications throughout Iowa State. Admintools was a plugin for Iowa State's Learning Management System, Blackboard. It provided functionality that would handle transfers of final grades from Blackboard to the legacy systems at the Registrar.

Technology Stack:

  • NIADS:
    Our stack primarily involved NetBSD servers that ran a C based web server. We loosely used CVS for commits, but also had some home-brew partitioning that allowed us to see our changes without truly deploying.
  • ITS Java Dev:
    Both projects Admintools and Harbor have used the Spring-boot stack. Admintools used a Java Server Pages (JSP) frontend, while Harbor used the react-redux framework. We used Git for version control. For the management side, we used primarily Atlassian products such as JIRA and BitBucket. We used deployment and software analytics tools SonarQube and Jenkins.

What I Learned:

  • NIADS:
    This team was fairly isolated. I learned a lot about how to do my own research and be an independent developer.
  • ITS Java Dev:
    This team used the Agile methodology. We used a Kanban board to track our issues and broke our tasks into 2-week sprints. I learned how to work at a steady pace, and became faster with my projects. I also learned how to work with a team, good code practices, and the Git workflow.
